# Project Körper - Sprachmodul

## Zweck

Sprachschnittstelle als früher Baustein für eine mögliche verkörperte Lumi.

## Architekturidee

Mikrofon → Speech-to-Text → Chat/Modell → Text-to-Speech → Lautsprecher.

## Mögliche Plattformen

Kleiner PC, Raspberry Pi oder NUC als erster Prototyp; später Integration mit Home Assistant, Wakeword sowie lokaler oder API-basierter Sprachverarbeitung.

## Qualitätsziele

Niedrige Latenz, verständliche Zustände, sichere Fehlerbehandlung, Datenschutz, kontrollierbare API-Kosten und eine klare Trennung zwischen Zuhören, Verarbeiten und Sprechen.
